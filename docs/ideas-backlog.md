# Ideas Backlog

Последнее обновление: 2026-05-20

Этот файл фиксирует продуктовые идеи до реализации. Он не заменяет DB-backed agent backlog, а хранит более крупные направления, которые нужно проработать перед кодом.

## Принцип продукта

Главная идея: приложение должно быть не просто списком карточек, а очередью внимания.

Пользователь открывает Mini App и быстро достаёт следующее напоминание из очереди. Если есть карточки, которые уже ждут оценки, они идут первыми. Если есть время, пользователь может вытаскивать будущие напоминания вперёд и разбирать их заранее.

## IDEA-001 Очередь Напоминаний

Статус: idea

Суть: первая дефолтная вкладка Mini App показывает не таблицу и не обычную ленту, а одну текущую карточку и действие `Достать следующую`.

Что должно попадать в очередь:
- напоминания, которые уже ждут оценки;
- просроченные напоминания;
- будущие напоминания по мере запроса пользователем.

Ожидаемый UX:
- пользователь видит одну крупную карточку;
- основные действия: `Окей`, `Снова`, `Перенести`, `Архив`;
- если карточки нет, главный сценарий переключается на быстрое создание напоминания;
- пользователь может попросить следующую карточку, когда у него есть время разбирать очередь наперёд.

Открытые вопросы:
- когда пользователь достал будущую карточку, она сразу становится `awaiting_grade` или нужен отдельный статус `viewed`;
- на сколько далеко вперёд можно вытаскивать напоминания;
- нужно ли отдельно показывать счётчики: ожидают оценки, просрочены, дальше по расписанию.

## IDEA-002 Быстрое Создание Напоминания

Статус: idea

Суть: создать напоминание из Mini App должно быть очень легко. По умолчанию это одноразовое напоминание, а не карточка обучения.

Ожидаемый UX:
- поле `Что напомнить?` сразу видно на первом экране;
- естественный язык: `завтра в 10 позвонить врачу`, `через 3 часа проверить деплой`;
- если время распознано, приложение показывает понятное подтверждение;
- если время не распознано, пользователь выбирает быстрый пресет или дату;
- доставка идёт туда же, куда сейчас приходят повторы.

Возможные быстрые пресеты:
- через час;
- сегодня вечером;
- завтра утром;
- выбрать дату и время.

V2:
- голосовое создание через кнопку микрофона;
- распознавание голосового текста в напоминание;
- подтверждение распознанного результата перед созданием.

Открытые вопросы:
- разрешать ли напоминание без времени;
- какие действия показывать после срабатывания: `Готово`, `Снова через час`, `Перенести`;
- должно ли быстрое напоминание уметь превращаться в обучающую карточку.

## IDEA-003 Групповые Очереди Через @бота

Статус: foundation

Суть: можно обращаться к боту через `@` в чатах, а к каждому чату привязывается собственная очередь напоминаний.

Зачем:
- создавать напоминания прямо в контексте группового обсуждения;
- вести отдельные очереди для разных чатов;
- в дальнейшем поддержать командные или семейные сценарии.

Открытые вопросы:
- нужен ли Mini App-переключатель между личной очередью и очередями чатов;
- кто может создавать и закрывать напоминания в групповом чате;
- где показывать очередь чата: в Mini App, в самом чате или в обоих местах;
- нужны ли роли владельца/участников очереди.

Foundation:
- `cards` и `reminder_jobs` имеют `queue_scope_type` / `queue_scope_id`;
- личная очередь остаётся `user:<user_id>`;
- групповая очередь создаётся как `chat:<telegram_chat_id>`;
- основной Telegram entrypoint для групп при выключенном privacy mode — `@BotUsername текст` или reply `@BotUsername`;
- сообщения группы, которые не начинаются с `@BotUsername`, должны отбрасываться до auth, DB writes, парсеров и app logs;
- explicit fallback остаётся `/add@BotUsername текст` или reply-команда `/add@BotUsername`.

## IDEA-004 Простые Курсы

Статус: spec

Суть: курс — это последовательность маленьких шагов, которые попадают в обычную очередь напоминаний по одному.

V1:
- курс создаётся вручную;
- курс состоит из шагов;
- шаг может быть материалом, практикой или вопросом для самопроверки;
- пользователь запускает курс себе;
- первый шаг попадает в очередь;
- после просмотра шага система готовит следующий шаг.

Не входит в v1:
- LLM-генерация;
- автопроверка ответов;
- маркетплейс курсов;
- отдельный LMS-интерфейс.

Подробнее: `docs/simple-courses-v1.md`.

## IDEA-005 Creator Подборки

Статус: idea

Суть: криейторы могут создавать обучающие подборки, а пользователь сохраняет их себе и проходит по расписанию.

Ожидаемый сценарий:
- автор создаёт подборку материалов;
- пользователь сохраняет подборку себе;
- система выдаёт материалы по очереди;
- прогресс сохраняется отдельно для каждого пользователя.

Открытые вопросы:
- публичные ли подборки или только по ссылке;
- нужны ли версии подборок;
- можно ли миксовать подборки с личными карточками пользователя.

## IDEA-006 Бэклог Идей В Mini App

Статус: idea

Суть: нужен фронт для backlog идей прямо в Mini App. Сначала только для владельца.

Ожидаемый UX:
- добавить идею текстом;
- посмотреть список идей;
- поменять статус;
- связать идею с карточкой, багом или будущей задачей;
- возможно отправлять идеи агенту как сейчас через Telegram backlog.

Открытые вопросы:
- это должен быть UI поверх `backlog_items` или отдельная таблица/тип;
- нужны ли статусы `idea`, `planned`, `in_progress`, `done`, `archived`;
- должен ли агент иметь read API для этого backlog.
